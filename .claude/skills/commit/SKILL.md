# コミットメッセージ生成スキル

変更内容を分析してコミットメッセージを生成する。

## 手順

### 1. 変更分析（内部処理、ユーザーに途中結果を見せない）

以下を内部的に実行し、コミットメッセージを生成する。ツールの実行結果はユーザーに見せない。

- `git status` と `git diff --stat` で変更内容を把握
- 変更がない場合は「コミットする変更がありません」と伝えて終了
- `docs/plans/` を参照してタスクを特定（特定できなければタスクなし）
- コミットメッセージを生成

### 2. コミットメッセージの提示

以下のフォーマットで生成したコミットメッセージを提示する。
**注意:** `git add` や `git commit` 等のコマンドは提示しない。メッセージのみ出力する。

```
<type>: <簡潔な要約（日本語、50文字以内）>

<本文（日本語、何をなぜ変更したか）>

refs: TASK-XXX（タスクが特定できた場合）
```

#### type

| type     | 用途             |
| -------- | ---------------- |
| feat     | 新機能           |
| fix      | バグ修正         |
| docs     | ドキュメント変更 |
| style    | フォーマット等   |
| refactor | リファクタリング |
| test     | テスト           |
| chore    | 設定・ビルド等   |

タスク完了の場合は以下も合わせて案内する。

```
タスク: TASK-XXX → ✅ 完了（プランのステータスを更新してください）
```

## $ARGUMENTS

- `/commit` - 通常のコミット
- `/commit メッセージの補足` - 補足情報を加味してメッセージ生成
