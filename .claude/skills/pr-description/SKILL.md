# PR概要生成スキル

現在のブランチのコミット履歴と差分を分析し、PR概要（タイトル・本文）を生成する。

## 手順

### 1. 変更分析（内部処理、ユーザーに途中結果を見せない）

以下を内部的に実行し、PR概要を生成する。

- `git branch --show-current` で現在のブランチ名を取得する
- ベースブランチを特定する（`main` とは限らない。リモート追跡ブランチや `git log --merges` 等からフォーク元を推定する。不明な場合はユーザーに確認する）
- `git log <base>..HEAD --oneline` でベースブランチからのコミット一覧を取得する
- `git diff <base>...HEAD --stat` で変更ファイルの統計を取得する
- `git diff <base>...HEAD` で差分の詳細を取得する
- コミットがない場合は「PRにする変更がありません」と伝えて終了する
- 関連するタスクファイル（`docs/tasks/TASK-XXX.md`）があれば参照する
- 関連する設計書（`docs/designs/`）があれば参照する

### 2. PRテンプレートの確認

- `.github/PULL_REQUEST_TEMPLATE.md` または `.github/pull_request_template.md` が存在する場合は、そのテンプレートに沿った内容で生成する
- テンプレートがない場合は、以下のデフォルトフォーマットで生成する

### 3. PR概要の生成

#### タイトル

```
<type>: <簡潔な要約（日本語、70文字以内）>
```

typeはcommitスキルと同じ規則に従う（feat / fix / docs / style / refactor / test / chore）。

#### 本文（デフォルトフォーマット）

```markdown
## 変更の概要

<この変更の目的と背景を簡潔に説明>

## 実装/修正内容

- <変更点1>
- <変更点2>
- ...

## 動作確認方法

- <確認手順1>
- <確認手順2>
- ...

## その他

- 関連タスク: TASK-XXX（該当する場合）
- 関連設計書: docs/designs/xxx（該当する場合）
```

### 4. 生成ルール

- コミット履歴と差分の両方から変更の意図を読み取る
- 「何を変えたか」だけでなく「なぜ変えたか」を記述する
- 動作確認方法は、レビュアーが実際に確認できる具体的な手順にする
- 設計書やタスクがある場合はリンクを含める
- 差分が大きい場合は、変更をカテゴリ別に整理する

### 5. ユーザーに確認

生成したPR概要（タイトル・本文）を提示し、承認を求める。

選択肢: 「OK」「修正を依頼する」「キャンセル」

### 6. 承認後の処理

ユーザーがOKしたら、PR概要を `/tmp/pr-description.md` に `Write` ツールで保存する。

```
✅ PR概要を生成しました: /tmp/pr-description.md
   タイトル: <PRタイトル>

PRを作成するには、以下をホスト側で実行してください:
  gh pr create --base <ベースブランチ> --title "<タイトル>" --body-file /tmp/pr-description.md
```

## $ARGUMENTS

- `/pr-description` - 現在のブランチからPR概要を生成
- `/pr-description 補足情報` - 補足情報を加味して生成
