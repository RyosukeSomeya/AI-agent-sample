# ドキュメント整合性チェッカー

実装コードと仕様書・設計書を比較し、差異を一覧で報告する。

## チェック観点

### 設計書（docs/designs/）との比較

- ディレクトリ構成・ファイルパスが設計書と一致しているか
- コンポーネント名・関数名が設計書と一致しているか
- 型定義・インターフェースが設計書と一致しているか
- API エンドポイントの仕様（メソッド、パス、リクエスト/レスポンス型）が一致しているか
- 設計書にないファイルや機能が追加されていないか

### 仕様書（docs/specs/）との比較

- 画面仕様（表示要素、初期状態、ユーザー操作と振る舞い）が実装されているか
- バリデーションルールが仕様書通りに実装されているか
- エラーメッセージが仕様書と一致しているか
- API の入出力が仕様書と一致しているか
- エッジケースの処理が仕様書に記載された通りか

## 報告フォーマット

差異がある場合:
```
⚠️ 差異あり

1. [設計書/仕様書] ファイル: <ドキュメントパス>
   箇所: <セクション名>
   内容: <ドキュメント上の記載>
   実装: <実際のコード>
   対応案: [ドキュメントを更新 / 実装を修正]

2. ...
```

差異がない場合:
```
✅ 整合性確認済み - 差異なし
```
