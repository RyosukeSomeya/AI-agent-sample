# 横断分析エージェント（crosscut）

## 学習テーマ

マルチエージェント構成における「専門エージェント」の設計パターンを学ぶ。

- **対応ワークショップ:** ワークショップ外 / 発展（マルチエージェント設計）
- **対応Step:** Step 3
- **対応タスク:** TASK-005

## このエージェントの役割

複数都市の気象データを受け取り、都市間の比較・相関分析・トレンド比較を行う。
オーケストレータ（TASK-006）から A2A 連携で呼び出される専門エージェント。

### 学習ポイント

1. **専門性はプロンプトで決まる**: analyst と同じツール構成（Code Interpreter + save_to_s3）だが、システムプロンプトで「横断比較」という役割を定義
2. **マルチエージェント設計思想**: 1つのエージェントに全部やらせるのではなく、専門性で分離する
3. **A2A連携の準備**: Agent インスタンスをオーケストレータのツールとして渡すことで、エージェント間通信が実現する

## 動作確認手順

```bash
cd packages/agents
uv run python -m crosscut
```

起動後、複数都市の気象データを含むプロンプトを入力する。

例:
```
東京（気温: 15°C, 湿度: 60%）と大阪（気温: 18°C, 湿度: 55%）のデータを比較分析してください
```

## 本番構成との差分

| 項目 | サンプル | 本番 |
|------|---------|------|
| 起動方法 | CLI対話 / オーケストレータから直接呼び出し | AgentCore Runtime (HTTPS) / A2A プロトコル |
| 分析対象 | 天気データ | 財務データ（分析パターンは同じ） |
| レポート保存先 | 1バケット（パスパターンで整理） | 複数バケットに分離 |
