# 異常検知エージェント（alert）

## 学習テーマ

閾値ベースのルールをシステムプロンプトで定義し、LLMが柔軟に判断する設計パターンを学ぶ。

- **対応ワークショップ:** ワークショップ外 / 発展（イベント駆動エージェント設計）
- **対応Step:** Step 3
- **対応タスク:** TASK-005

## このエージェントの役割

気象データを監視し、急激な変化や危険な状況を検知してアラートJSON を生成する。
本番構成の「シグナル検知エージェント」に相当する。

### 学習ポイント

1. **ルールをプロンプトで定義**: 従来のハードコードされた if 文ではなく、システムプロンプトにルールを記述し LLM が柔軟に判断する
2. **構造化出力**: アラートを JSON 形式で出力し、後続システム（SNS 通知、S3 保存）と連携しやすくする
3. **イベント駆動の準備**: Step 7 以降で EventBridge イベントをトリガーに自動起動される構成に発展する

### 検知ルール

| 条件 | アラートタイプ | 重要度目安 |
|------|---------------|-----------|
| 24時間以内の気温変化が10°C以上 | temperature_change | warning |
| 風速が15m/s以上 | strong_wind | warning〜critical |
| 降水量が50mm/h以上 | heavy_rain | critical |
| 災害警報が発表中 | disaster | critical |

## 動作確認手順

```bash
cd packages/agents
uv run python -m alert
```

起動後、気象データを含むプロンプトを入力する。

例:
```
東京の気象データ: 昨日の気温 25°C → 今日 13°C（12°C低下）、風速 8m/s、降水量 5mm/h
```

→ LLM が「24時間以内の気温変化が10°C以上」ルールに基づき、アラートJSON を生成する。

## 本番構成との差分

| 項目 | サンプル | 本番 |
|------|---------|------|
| 起動方法 | CLI対話 / オーケストレータから呼び出し | EventBridge イベントトリガー |
| 検知対象 | 天気データ | 財務異常シグナル |
| アラート通知 | コンソール出力 + S3保存 | SNS → Slack / メール通知 |
| ルール管理 | システムプロンプトに記述 | Guardrails + プロンプト |
